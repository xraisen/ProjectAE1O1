Vending Canada
Attribution Stack

A centralized implementation designed to solve attribution decay for high-intent vending service leads. This stack ensures GCLIDs and UTMs are preserved even if a user browses for days before inquiring.

Integration Goals

Persistence

Uses dual LocalStorage + Cookies. UTMs survive ITP restrictions and multi-day consideration cycles.

Auto-Inject

Automatically populates Fluent Form hidden fields (gclid, utm_service) for WP Entry exports.

EC Privacy

SHA-256 Hashing for Email addresses allows privacy-safe Google Ads matching (Enhanced Conversions).

Dynamic

MutationObservers track forms in AJAX modals, Divi Overlays, and dynamic page loads.

VendingCanada Implementation Protocol

01

One Master Trigger

Do NOT use the "All Forms" trigger. Use a Custom Event trigger named FluentFormActivities with the condition {{FluentFormEventAction}} equals FormSubmitted.

02

WP Hidden Field Mapping

In the Fluent Form editor, add hidden fields. Set the Name Attribute (found in Advanced options) to exactly: gclid, utm_service, and utm_source.

03

Enhanced Conversions Setup

The Google Ads User-Provided Data Event tag must use the sha256_email_address variable. The script handles the hashing automatically via the browser's crypto API.
